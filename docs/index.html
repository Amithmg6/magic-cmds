magic-cmds

Jupyter Notebook (and IPython) magic commands are powerful, non-standard commands that extend the functionality of the notebook environment beyond pure Python code. They are prefixed with a single percent sign (%) for line magics (operating on a single line) or a double percent sign (%%) for cell magics (operating on the entire cell). These commands streamline workflows, enhance interactivity, and provide direct access to system operations, debugging tools, and more.

I. Information and Help

These commands help you discover and understand other magic commands and IPython features.

%lsmagic

Lists all available magic commands (both line and cell magics). This is often the first command to run when exploring magic functionalities.

# No specific Python code to run with %lsmagic itself, as it lists magics.
%lsmagic

%quickref

Displays a quick reference guide to IPython's features, including a summary of common magic commands.

# No specific Python code to run with %quickref itself.
%quickref

%magic

Provides a more general description of available magic functions, with some examples.

# No specific Python code to run with %magic itself.
%magic

%<command>?

Appending % to any magic command (e.g., %timeit?) displays its docstring, providing detailed information about its usage, arguments, and options.

# Get help on the %timeit magic command
%timeit?

%<command>??

Appending ?? to a magic command displays its source code.

# View the source code of the %lsmagic command
%lsmagic??

II. Execution and Timing

These commands are crucial for profiling and optimizing your code, helping you understand execution speed.

%time

Measures the execution time of a single Python statement or expression.

# Time a simple list creation
my_list = list(range(100000))
%time sum(my_list)

%%time

Measures the execution time of the entire cell.

%%time
# Multi-line code to be timed
import numpy as np
data = np.random.rand(1000, 1000)
result = np.linalg.det(data)

%timeit

Measures the execution time of a single line of code multiple times (typically 7 runs with 1,000 to 100,000 loops each) to get a more accurate average.

# Compare performance of list comprehension vs map
%timeit [x**2 for x in range(1000)]
%timeit list(map(lambda x: x**2, range(1000)))

%%timeit

Measures the execution time of the entire cell multiple times for more robust results.

%%timeit
# Define and call a function within the timed block
def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a
fibonacci(20)

%prun

Runs a statement through the Python code profiler (cProfile). Useful for identifying performance bottlenecks within your code.

# Define a function with a potential bottleneck
def slow_function():
    total = 0
    for i in range(10000):
        for j in range(1000):
            total += i * j
    return total

%prun slow_function()

%run

Executes a Python script (.py file) within the IPython environment. Any functions or variables defined in the script become available in your current session.

# First, create a dummy Python file named 'my_script.py'
# %%writefile my_script.py
# def greet(name):
#     print(f"Hello, {name}!")
# greet("World")

# Then, run the script
%run my_script.py
# Now, the 'greet' function is available in the notebook
greet("Jupyter User")

III. Environment and Workspace Management

These commands help you manage your Python environment and the Jupyter Notebook's state, including variables and history.

%who

Lists all interactive variables (names only) currently defined in the namespace.

a = 10
b = "hello"
my_list = [1, 2, 3]
%who

%whos

Lists all interactive variables with more details, including their type and value.

a = 10
b = "hello"
my_list = [1, 2, 3]
%whos

%who_ls

Returns a sorted list of all interactive variables.

x = 1
y = 2
z = 3
%who_ls

%xdel

Deletes a variable and attempts to clear it from IPython's machinery, useful for freeing up memory.

import numpy as np
large_array = np.random.rand(10000, 10000)
print(f"Size of large_array: {large_array.nbytes / (1024**2):.2f} MB")
%xdel large_array
# large_array is now deleted and memory freed

%reset

Resets the entire IPython namespace, deleting all variables and imported modules. Use with caution! Can be used with -f to force without confirmation.

# Define some variables
var1 = 10
var2 = "test"
import pandas as pd
df = pd.DataFrame({'col': [1,2]})

# Reset the namespace (this will clear var1, var2, df, and pandas import)
%reset -f
# Now, trying to access var1 will result in an error
# print(var1) # This would raise NameError

%hist / %history

Shows the history of executed commands. You can specify arguments like -n for the last n commands.

# Run a few commands first
print("Hello")
x = 5
y = x * 2

# View the last 3 commands in history
%history -n 3

%env

Displays, sets, or deletes environment variables.

# Display current environment variables
%env

# Set a new environment variable
%env MY_CUSTOM_VAR=my_value

# Access the variable in Python
import os
print(os.environ.get("MY_CUSTOM_VAR"))

%automagic

Toggles whether magic commands require the leading % for line magics. When "on," you can type timeit instead of %timeit. (Default is usually "on" in Jupyter).

# Check current automagic state
%automagic

# Toggle automagic off
%automagic off
# Now you'd need to use %timeit explicitly
# timeit [1,2,3] # This would fail

# Toggle automagic on again
%automagic on
# Now you can use timeit without % again
# timeit [1,2,3] # This would work

IV. File and System Operations

These commands allow you to interact with the file system directly from your notebook, similar to shell commands.

%cd

Changes the current working directory.

# Print current directory
%pwd
# Change to a different directory (e.g., a subfolder)
# %cd my_subfolder
# Change back to the parent directory
%cd ..
%pwd

%pwd

Prints the current working directory.

%pwd

%ls

Lists the contents of the current directory.

# List all files and directories
%ls
# List with long format (details)
%ls -l

%cat

Displays the content of a file.

# Create a dummy file first
# %%writefile test_file.txt
# Hello, this is a test file.
# It has multiple lines.

# Display its content
%cat test_file.txt

%cp

Copies files. (Similar to shell cp)

# Create source file
# %%writefile source.txt
# This is the source content.

# Copy it to a new file
%cp source.txt destination.txt
%ls

%mv

Moves/renames files. (Similar to shell mv)

# Create a file to rename
# %%writefile old_name.txt
# Content of the old file.

# Rename the file
%mv old_name.txt new_name.txt
%ls

%rm

Removes files. (Similar to shell rm)

# Create a file to remove
# %%writefile to_delete.txt
# This file will be deleted.

# Remove the file
%rm to_delete.txt
%ls

%mkdir

Creates a directory. (Similar to shell mkdir)

%mkdir new_folder
%ls

%rmdir

Removes an empty directory. (Similar to shell rmdir)

# Create and then remove an empty folder
%mkdir empty_folder
%rmdir empty_folder
%ls

%load

Loads the contents of an external file into the current cell.

# Create a Python script
# %%writefile my_module.py
# def calculate_square(x):
#     return x * x
# print("Module loaded!")

# Load the script into the current cell (it will appear as code)
%load my_module.py
# After loading, you can execute the cell to define the function
# print(calculate_square(5))

%%writefile

Writes the contents of the current cell to an external file. If the file exists, it overwrites it unless -a (append) is used.

%%writefile my_output.txt
This is the first line.
This is the second line.
# Check the file content using %cat or your file system after running this cell.

%pycat

Displays the syntax-highlighted content of a Python file.

# Create a Python file first
# %%writefile simple_func.py
# def add_numbers(a, b):
#     """Adds two numbers."""
#     return a + b

# Display its content with syntax highlighting
%pycat simple_func.py

V. Display and Output Control

These commands influence how output is displayed in the notebook, especially for visualizations and rich media.

%matplotlib inline

(Very common for data visualization) Ensures that Matplotlib plots are displayed directly within the notebook cells.

import matplotlib.pyplot as plt
import numpy as np

%matplotlib inline
x = np.linspace(0, 10, 100)
y = np.sin(x)
plt.plot(x, y)
plt.title("Sine Wave (Inline)")
plt.show()

%matplotlib notebook

Provides interactive Matplotlib plots within the notebook (allowing zooming, panning, etc.).

import matplotlib.pyplot as plt
import numpy as np

%matplotlib notebook
x = np.linspace(0, 10, 100)
y = np.cos(x)
plt.plot(x, y)
plt.title("Cosine Wave (Interactive)")
plt.show()

%%capture

Runs the cell but captures all its output (stdout, stderr, rich display calls), preventing it from being displayed. Useful for suppressing verbose output or for programmatic handling of output.

%%capture captured_output
# This print statement's output will be captured
print("This message will not appear directly in the output cell.")
x = 10 + 20
print(f"The value of x is {x}")

# You can access the captured output later
# captured_output.stdout
# captured_output.stderr
# captured_output.outputs

VI. Debugging

Commands to help you debug your Python code within the notebook environment.

%debug

Activates the interactive debugger when an exception occurs.

def divide_by_zero(a, b):
    return a / b

# Run the magic command first, then the problematic code
%debug
divide_by_zero(10, 0) # This will trigger the debugger on ZeroDivisionError

%pdb

Controls the automatic loading of the Python debugger.

# Turn on automatic pdb
%pdb on

def problematic_function():
    x = 10
    y = 0
    z = x / y # This will automatically drop into pdb

problematic_function()

# Turn off automatic pdb
%pdb off

VII. External Language Execution

These cell magics allow you to execute code in other languages directly within a Jupyter cell, provided the necessary interpreter is installed and configured.

%%bash / %%sh

Runs the cell content as a Bash or shell script.

%%bash
# List files in the current directory
ls -lh

# Print a message
echo "Hello from Bash!"

# Run a Python command from bash
python -c "print('Python from Bash')"

%%perl

Runs the cell content as Perl code.

%%perl
# Perl code example
my $name = "World";
print "Hello, $name from Perl!\n";
for (my $i = 0; $i < 3; $i++) {
    print "Loop iteration: $i\n";
}

%%ruby

Runs the cell content as Ruby code.

%%ruby
# Ruby code example
puts "Hello from Ruby!"
1.upto(3) do |i|
  puts "Ruby loop: #{i}"
end

%%python / %%python2 / %%python3

Explicitly runs the cell content as Python code (useful if you're in a multi-kernel environment or want to specify a Python version).

%%python
# This is standard Python code, explicitly run by the Python interpreter
def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

print(f"Factorial of 5 is: {factorial(5)}")

%%script

Allows executing a script using a specified interpreter.

%%script python
# This cell will be executed by the 'python' interpreter
import sys
print(f"Running with Python interpreter: {sys.version.splitlines()[0]}")
print("This is a script executed via %%script.")

%%sx

Executes a shell command and captures its output.

%%sx ls -la
# The output of 'ls -la' will be captured and can be accessed as a list of strings
# For example, to print the first line of output:
# print(_sx[0])

VIII. Miscellaneous / Advanced

A collection of other useful and more advanced magic commands.

%alias

Defines an alias for a system command.

# Define 'll' as an alias for 'ls -l'
%alias ll ls -l
# Now you can use 'll' directly
ll

%alias_magic

Creates an alias for an existing line or cell magic.

# Create 'tm' as an alias for '%timeit'
%alias_magic tm timeit
# Now you can use 'tm' instead of '%timeit'
tm [x for x in range(100)]

%bookmark

Manages IPython's bookmark system for directories.

# Go to a directory
# %cd /tmp

# Bookmark the current directory as 'temp_dir'
%bookmark temp_dir

# Change to another directory
# %cd /usr/local

# Jump back to the bookmarked directory
%cd -b temp_dir
%pwd

%connect_info

Displays connection information for the current kernel.

%connect_info

%config

Allows you to configure IPython and Jupyter components.

# Configure Matplotlib to use inline backend by default
%config InlineBackend.figure_format = 'retina'
%config IPKernelApp.pylab='inline' # Ensure inline plotting is enabled

%dhist

Displays the directory history.

# Change directories a few times to build history
# %cd /tmp
# %cd /var
# %cd ~

# Display directory history
%dhist

%edit / %ed

Opens an external text editor with the specified content (e.g., a function definition or history lines).

# Define a simple function
def my_function(x):
    return x * 2

# Open it in an external editor (requires editor configuration)
# %edit my_function

%gui

Enables GUI event loop integration for various GUI toolkits (e.g., qt, tk, wx, gtk3, glut).

# Example for PyQt (requires PyQt to be installed)
# import sys

%killbgscripts

Kills all background scripts launched by the %%script magic.

# Example: Run a script in the background (this will usually block the cell)

# To stop it:

%load_ext

Loads an IPython extension by its module name. Many powerful functionalities can be added this way.

# Load the autoreload extension for automatic code reloading
%load_ext autoreload
%autoreload 2 # Enable autoreload for all modules

%logoff / %logon / %logstart / %logstate

Commands for managing IPython's logging system.

# Start logging to a file

# Perform some actions
x = 10
print(x)

# Check log state
%logstate

# Turn off logging
%logoff

Pastes code from the clipboard, handling common indentation issues.

%pinfo / %pinfo2 / %pfile / %pdef / %pdoc / %psource

A suite of commands for inspecting objects, functions, and modules, providing information like docstrings, file locations, definitions, and source code.

# Get information about a built-in function
%pinfo list.append

# Get source code of a function
def my_custom_func(x):
    """This is a custom function."""
    return x * 10
%psource my_custom_func

%popd / %pushd

Manipulate the directory stack (similar to shell pushd/popd).

# Push current directory to stack and change to /tmp
%pushd /tmp
%pwd

# Push /var to stack and change to it
%pushd /var
%pwd

# Pop /var from stack, return to /tmp
%popd
%pwd

# Pop /tmp from stack, return to original directory
%popd
%pwd

%reload_ext

Reloads an IPython extension.

# If you modified an extension, you can reload it without restarting the kernel
# %reload_ext my_custom_extension

%unalias

Removes an alias.

# Define an alias first
%alias myls ls -l

# Check if it exists
%alias

# Remove the alias
%unalias myls
# Now 'myls' will no longer work

%xmode

Controls the verbosity of exception traceback.

# Set xmode to 'Plain' for concise tracebacks
%xmode Plain

# Trigger an error
1 / 0

# Set xmode back to 'Verbose' for full tracebacks
%xmode Verbose
1 / 0
